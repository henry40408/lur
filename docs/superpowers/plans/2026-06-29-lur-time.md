# lur.time Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a pure-compute `lur.time` capability: millisecond wall-clock (`now_ms`) and monotonic (`monotonic_ms`) clocks, plus `parse_rfc3339`/`parse_http_date` that turn timestamp strings into epoch milliseconds.

**Architecture:** A new `src/capabilities/time.rs` submodule following the `base64.rs`/`crypto.rs` shape: an `install(lua, lur)` registering a flat `lur.time` table of `create_function` closures. Clocks use std `SystemTime`/`Instant`; parsing reuses the existing `chrono` dependency (RFC 3339) and promotes the already-in-tree `httpdate` crate to a direct dependency (HTTP-date). Wired into `capabilities::install` immediately after `cookie`.

**Tech Stack:** Rust (edition 2024), mlua 0.11 (Luau), chrono 0.4 (already a dependency), httpdate 1.0.3 (promoted from transitive to direct).

## Global Constraints

- Pure-compute capability — **not** policy-gated (no `Arc<Policy>` arg). Installed in `capabilities::install` immediately after `cookie::install`, before `sandbox(true)`.
- **Unit is integer milliseconds (`i64`)** for every value produced or accepted.
- Errors raise `lur.time.<fn>: <detail>` via `mlua::Error::runtime`, catchable with `pcall`.
- `now_ms` = `SystemTime::now()` − `UNIX_EPOCH` in ms; a pre-epoch clock raises. `monotonic_ms` = ms since a process-fixed `LazyLock<Instant>`, captured once; only differences are meaningful. `parse_rfc3339` uses `chrono::DateTime::parse_from_rfc3339` → `timestamp_millis()`. `parse_http_date` uses `httpdate::parse_http_date` → ms from `UNIX_EPOCH`.
- No timestamp **formatting** (out of scope — `os.date("!…")` already covers it).
- **Dependency policy:** `httpdate` is pinned to `1.0.3` — the exact version already resolved in `Cargo.lock` (pulled in via hyper), so promoting it to a direct dependency adds **no** new code to the tree. `1.0.3` is long-published (satisfies the 7-day cooldown); `cargo deny check` must stay green.
- Tests live in `tests/capabilities.rs` using the existing `run(src)` helper (matches the `crypto`/`cookie` convention; `run` is `Runtime::new().run(src)`, which drives a current-thread tokio runtime so `lur.async.sleep` works inside it).
- CI gates: `cargo fmt --all`, `cargo clippy --all-targets -- -D warnings`, `cargo nextest run`, `cargo deny check`. Run `cargo fmt --all` before every commit.
- Commits GPG-signed (`git commit -S`). Stage files explicitly by name — never `git add -A`/`.`.

```rust
//! `lur.time` — millisecond clocks and timestamp parsing. Pure-compute
//! capability, no policy gate. Fills the gaps Luau's `os.*` cannot: sub-second
//! and monotonic timing, and parsing RFC 3339 / HTTP-date strings into numbers.
//! Formatting stays with `os.date` (which already emits both). All values are
//! integer milliseconds.

use std::sync::LazyLock;
use std::time::{Instant, SystemTime, UNIX_EPOCH};

use mlua::{Error, Lua, Table};

use crate::runtime::RunError;

/// Process-fixed reference for `monotonic_ms`, captured on first use. Only the
/// difference between two readings is meaningful.
static MONOTONIC_START: LazyLock<Instant> = LazyLock::new(Instant::now);

/// Install the flat `lur.time` table.
pub fn install(lua: &Lua, lur: &Table) -> Result<(), RunError> {
    let time = lua.create_table().map_err(RunError::Init)?;

    install_clocks(lua, &time)?;

    lur.set("time", time).map_err(RunError::Init)?;
    Ok(())
}

/// `lur.time.now_ms` / `lur.time.monotonic_ms`.
fn install_clocks(lua: &Lua, time: &Table) -> Result<(), RunError> {
    let now_ms = lua
        .create_function(|_, ()| {
            let dur = SystemTime::now().duration_since(UNIX_EPOCH).map_err(|_| {
                Error::runtime("lur.time.now_ms: system clock is before the unix epoch")
            })?;
            Ok(dur.as_millis() as i64)
        })
        .map_err(RunError::Init)?;
    time.set("now_ms", now_ms).map_err(RunError::Init)?;

    let monotonic_ms = lua
        .create_function(|_, ()| Ok(MONOTONIC_START.elapsed().as_millis() as i64))
        .map_err(RunError::Init)?;
    time.set("monotonic_ms", monotonic_ms)
        .map_err(RunError::Init)?;

    Ok(())
}
```

```rust
/// `lur.time.parse_rfc3339` / `lur.time.parse_http_date`.
fn install_parsers(lua: &Lua, time: &Table) -> Result<(), RunError> {
    let parse_rfc3339 = lua
        .create_function(|_, s: mlua::String| {
            let s = s
                .to_str()
                .map_err(|e| Error::runtime(format!("lur.time.parse_rfc3339: {e}")))?;
            let dt = chrono::DateTime::parse_from_rfc3339(&s)
                .map_err(|e| Error::runtime(format!("lur.time.parse_rfc3339: {e}")))?;
            Ok(dt.timestamp_millis())
        })
        .map_err(RunError::Init)?;
    time.set("parse_rfc3339", parse_rfc3339)
        .map_err(RunError::Init)?;

    let parse_http_date = lua
        .create_function(|_, s: mlua::String| {
            let s = s
                .to_str()
                .map_err(|e| Error::runtime(format!("lur.time.parse_http_date: {e}")))?;
            let t = httpdate::parse_http_date(&s)
                .map_err(|e| Error::runtime(format!("lur.time.parse_http_date: {e}")))?;
            let dur = t.duration_since(UNIX_EPOCH).map_err(|_| {
                Error::runtime("lur.time.parse_http_date: date is before the unix epoch")
            })?;
            Ok(dur.as_millis() as i64)
        })
        .map_err(RunError::Init)?;
    time.set("parse_http_date", parse_http_date)
        .map_err(RunError::Init)?;

    Ok(())
}
```
